GSoC Week: 1

Hello World! This series of blogs is about how I pull my hairs out to complete the Gsoc Project of adding support for Matrix to Instantbird. You can read the abstract for the project here, and the complete project proposal I then created tabtest.xml and tabtest.css (for binding) in order to test it out.here[link to be updated].

Well, the plan for the first two weeks was to implement Direct Conversations. Matrix manages direct rooms in a slightly different way from Instantbird. Matrix differentiates Direct and Group Chats by flagging DM Rooms with "is_direct", allows to open DM conversations with same interlocutor and allow inviting others users to DMs. Whereas Instantbird has different classes for DMs and Group Chats, doesn't allow multiple DMs with same user, and no inviting other users to Direct Chats. So, all I have do is to make both of them work, without breaking any of them.

The first few days went experimenting and creating different rooms with different matrix users(thereby spamming everyone with invites) Creating a Direct Conversation Room involves mainly two things. 1. creating the room with createRoom({is_direct: true}) 2. get/set the accountDate and update the Dm
